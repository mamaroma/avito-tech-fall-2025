PR Reviewer Service

Сервис для автоматического назначения ревьюверов на Pull Request’ы внутри команды.

Возможности:
- создание команд;
- создание пользователей и управление активностью;
- создание Pull Request;
- автоматическое назначение до двух активных ревьюверов из команды автора (автор исключается);
- переназначение ревьювера на случайного активного участника команды заменяемого;
- запрет изменения ревьюверов после merge;
- идемпотентный merge;
- получение списка PR, назначенных выбранному ревьюверу.

Архитектура проекта:
- cmd/server — точка входа приложения;
- internal/app/http — обработчики HTTP-эндпоинтов;
- internal/app/services — бизнес-логика;
- internal/repository/postgres — слой доступа к данным;
- internal/domain — доменные сущности и ошибки;
- migrations — SQL-миграции;
- deployments — Dockerfile и docker-compose окружение;
- tests — unit-тесты сервисного слоя.

Запуск сервиса:
1. Перейти в каталог deployments
2. Выполнить команду:
   docker-compose up --build
3. Сервис будет доступен на порту 8080.
4. Миграции применяются автоматически.

Переменные окружения:
- DB_HOST
- DB_PORT
- DB_USER
- DB_PASSWORD
- DB_NAME

Тестирование:
- тесты находятся в каталоге tests;
- запуск тестов:
  go test ./...

Основные допущения:
- в небольших командах после переназначения количество ревьюверов может уменьшаться, что соответствует требованиям;
- автор PR всегда исключается из кандидатов при назначении и переназначении;
- при отсутствии доступных кандидатов возвращается ошибка;
- эндпоинт получения PR по ревьюверу возвращает только основную информацию о PR, без полного списка ревьюверов.
